Yes, tracking. You want **Spotlight Comments** — the field built into Get Info labeled "Comments". Apple has had this forever and most people ignore it.

How it works:
- Right-click a file, Get Info, scroll to "Comments". Paste your paragraph there.
- You can add Comments as a Finder column: View menu, Show View Options, check "Comments".
- Spotlight indexes it, so you can search the paragraph text and the file comes up.
- Programmatically it is `kMDItemFinderComment`, settable via AppleScript or xattr.

The catch: Finder comments can get stripped when files move across file systems, especially onto FAT drives or some network shares. Inside APFS and iCloud they survive. Google Drive sync is a known weak spot, so if the raw video lives in Drive, the comment may not round trip.

My recommendation: belt and suspenders. Finder comment for the human eye in Get Info and Finder columns, plus a sidecar markdown file next to the video with the same summary plus full metadata. The sidecar is Claude readable, grep able, and survives anywhere.

Yes, tracking. You want **Spotlight Comments** — the field built into Get Info labeled "Comments". Apple has had this forever and most people ignore it.

How it works:
- Right-click a file, Get Info, scroll to "Comments". Paste your paragraph there.
- You can add Comments as a Finder column: View menu, Show View Options, check "Comments".
- Spotlight indexes it, so you can search the paragraph text and the file comes up.
- Programmatically it is `kMDItemFinderComment`, settable via AppleScript or xattr.

The catch: Finder comments can get stripped when files move across file systems, especially onto FAT drives or some network shares. Inside APFS and iCloud they survive. Google Drive sync is a known weak spot, so if the raw video lives in Drive, the comment may not round trip.

My recommendation: belt and suspenders. Finder comment for the human eye in Get Info and Finder columns, plus a sidecar markdown file next to the video with the same summary plus full metadata. The sidecar is Claude readable, grep able, and survives anywhere.

LFS stands for Large File Storage. It is Git's extension for handling big binary files like videos. Instead of storing the actual file in the repo, Git stores a tiny pointer, and the real file lives on a separate LFS server.

Yes, tracking. You want **Spotlight Comments** — the field built into Get Info labeled "Comments". Apple has had this forever and most people ignore it.

How it works:
- Right-click a file, Get Info, scroll to "Comments". Paste your paragraph there.
- You can add Comments as a Finder column: View menu, Show View Options, check "Comments".
- Spotlight indexes it, so you can search the paragraph text and the file comes up.
- Programmatically it is `kMDItemFinderComment`, settable via AppleScript or xattr.

The catch: Finder comments can get stripped when files move across file systems, especially onto FAT drives or some network shares. Inside APFS and iCloud they survive. Google Drive sync is a known weak spot, so if the raw video lives in Drive, the comment may not round trip.

My recommendation: belt and suspenders. Finder comment for the human eye in Get Info and Finder columns, plus a sidecar markdown file next to the video with the same summary plus full metadata. The sidecar is Claude readable, grep able, and survives anywhere.
